Editorial Policy편집 정책

How NEONCPS decides what belongs on the site무엇이 사이트에 올라갈 가치가 있는지 NEONCPS가 판단하는 기준

This policy explains the review standard behind NEONCPS pages: source quality, update triggers, correction handling, and the rule that ads or UI must never pretend to be navigation or downloads.

이 문서는 NEONCPS 페이지의 검토 기준을 설명합니다. 출처 품질, 업데이트 기준, 정정 처리, 그리고 광고나 UI가 탐색이나 다운로드처럼 위장하면 안 된다는 원칙을 다룹니다.

A utility page needs clarity plus honest limits유틸리티 페이지에는 명확함과 정직한 한계 설명이 필요합니다

NEONCPS does not treat a page as complete just because the button works. If the score is easy to misuse, the surrounding explanation is part of the product. If the explanation becomes stale, the page loses value even if the code still runs.

버튼이 작동한다고 해서 페이지가 완성됐다고 보지 않습니다. 점수가 오해되기 쉽다면 주변 설명도 제품의 일부입니다. 설명이 낡으면 코드가 살아 있어도 페이지의 가치는 떨어집니다.

Core review questions핵심 검토 질문

  • Does the text match the live tool?
  • 문서가 실제 도구와 맞는가?
  • Does the page tell the visitor something specific beyond the score?
  • 점수 외에 구체적인 설명을 제공하는가?
  • Is the navigation honest and non-deceptive?
  • 탐색 구조가 정직하고 기만적이지 않은가?

Update triggers갱신 트리거

  • Logic change in the tool.
  • 도구 로직 변경.
  • Broken links or outdated claims.
  • 깨진 링크나 오래된 주장.
  • Reports that wording is misleading or incomplete.
  • 문장이 오해를 부르거나 부족하다는 제보.

Source standard출처 기준

Pages about CPS methodology, device behavior, fatigue, timing, or safety should be grounded in one or more of the following: the live NEONCPS code path, web standards and platform documentation, official policy documentation, or credible research relevant to repeated finger tapping, input latency, or repetitive use.

CPS 측정 원리, 기기 동작, 피로, 타이밍, 안전성을 설명하는 페이지는 다음 중 하나 이상에 근거해야 합니다. 실제 NEONCPS 코드 경로, 웹 표준과 플랫폼 문서, 공식 정책 문서, 반복 태핑·입력 지연·반복 사용과 관련된 신뢰 가능한 연구 자료입니다.

Update rule업데이트 규칙

  • When the page changes how it counts input, timing, or result display, related explanatory text should be reviewed in the same cycle.
  • 입력 집계, 시간 계산, 결과 표시가 바뀌면 관련 설명도 같은 주기 안에서 검토해야 합니다.
  • If navigation changes, trust statements and policy pages should be rechecked as well.
  • 탐색 구조가 바뀌면 신뢰 문구와 정책 페이지도 같이 다시 확인해야 합니다.
  • If an article becomes shallow, redundant, or detached from the utility, it should be rewritten or removed.
  • 문서가 얕아지거나 중복되거나 도구와 분리되면 다시 쓰거나 제거해야 합니다.

Correction handling정정 처리

NEONCPS accepts correction requests through the home-page contact form. The intended flow is simple: verify the report, identify the affected claim, revise the page so it matches reality, and keep the fix local to the claim that was wrong.

NEONCPS는 홈 페이지 문의 폼을 통해 정정 요청을 받습니다. 절차는 단순합니다. 제보를 확인하고, 영향을 받는 문장을 찾고, 현실과 맞도록 페이지를 고치고, 잘못된 주장이 있던 위치를 바로잡습니다.

User experience and ad rule사용자 경험과 광고 원칙

Ads, buttons, or cards must never imitate site navigation, fake downloads, system warnings, or hidden product steps. The main interactive elements should remain obviously functional.

광고, 버튼, 카드가 사이트 탐색, 가짜 다운로드, 시스템 경고, 숨겨진 단계처럼 보이게 해서는 안 됩니다. 주요 상호작용 요소는 항상 실제 기능으로 보여야 합니다.

  • No buttons that suggest installation when nothing is being installed.
  • 설치가 없는데 설치처럼 보이게 하는 버튼을 만들지 않습니다.
  • No links that force extra clicks without adding information.
  • 정보 없이 클릭만 강요하는 링크를 만들지 않습니다.
  • No ad styling that looks like the next required step of the tool.
  • 광고가 도구의 다음 단계처럼 보이게 꾸미지 않습니다.
  • No empty archive or category pages for keywords only.
  • 키워드만 위한 빈 카테고리나 아카이브 페이지를 만들지 않습니다.

Thin-content check for new pages새 페이지의 빈약한 콘텐츠 점검 기준

Question질문 Required answer필수 기준
Does the page solve a real visitor question tied to the tool?이 페이지가 도구와 연결된 실제 질문을 해결하는가? It should answer methodology, score variance, device difference, or safe use.측정 원리, 점수 변동, 기기 차이, 안전한 사용 같은 구체적 질문에 답해야 합니다.
Is the wording specific to NEONCPS?문장이 NEONCPS에 특화돼 있는가? It should mention the real score rule or interpretation standard used here.실제 점수 계산 규칙이나 이 사이트의 해석 기준이 드러나야 합니다.
Would the page still be worth reading without ads?광고가 없어도 읽을 가치가 있는가? If not, it is not ready.그렇지 않다면 아직 준비되지 않은 페이지입니다.

A page should help the user understand the tool more accurately than the number alone. If it does not do that, it is not ready to stay.

페이지는 숫자 하나보다 더 정확하게 도구를 이해하도록 도와야 합니다. 그 역할을 못 한다면 남아 있을 준비가 된 것이 아닙니다.

Site purpose is summarized on the About page, and privacy details stay on the Privacy page.

사이트 목적은 소개 페이지에, 개인정보 세부 내용은 개인정보 페이지에 정리되어 있습니다.
