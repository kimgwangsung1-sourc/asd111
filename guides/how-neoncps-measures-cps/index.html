How NEONCPS measures CPSNEONCPS가 CPS를 재는 방법

This page explains what is counted, what is excluded, how time is measured, and why the final number is best used for repeat runs in the same setup.

The score is simple on purpose점수는 의도적으로 단순합니다

NEONCPS calculates CPS as total counted inputs divided by the selected duration. The simplicity is intentional. A short browser utility is most useful when the visitor can see the rule, repeat the test, and understand the result without hidden weighting.

NEONCPS는 선택한 시간 동안 집계된 총 입력 수를 시간으로 나누어 CPS를 계산합니다. 이 단순함은 의도된 것입니다. 짧은 브라우저 유틸리티는 규칙이 보이고, 반복할 수 있고, 숨은 가중치 없이 결과를 이해할 수 있을 때 가장 유용합니다.

Core formula기본 공식

CPS = total counted inputs / selected seconds

Input rule입력 기준

Primary pointer input during the active window. Right-click is ignored.

측정 구간의 기본 포인터 입력만 집계하고 우클릭은 제외합니다.

1. What gets counted1. 무엇을 세는가

The page uses a narrow rule: count the primary click or tap events that happen during the selected test duration. That rule keeps desktop and mobile comparisons cleaner and avoids score inflation from inputs that do not represent the same action.

이 페이지는 좁은 기준을 사용합니다. 선택한 시간 동안 발생한 기본 클릭 또는 탭만 세는 방식입니다. 이렇게 해야 데스크톱과 모바일 비교가 조금 더 깔끔해지고, 같은 동작이 아닌 입력으로 점수가 부풀려지는 일을 줄일 수 있습니다.

  • Primary pointer input is the target because it matches the intended click or tap motion.
  • 기본 포인터 입력은 의도한 클릭/탭 동작과 가장 잘 맞기 때문에 기준이 됩니다.
  • Right-click is excluded because it is not part of the intended comparison rule.
  • 우클릭은 비교 기준에 포함되지 않으므로 제외됩니다.
  • The explanatory copy is kept aligned with the real score rule.
  • 설명 문구는 실제 점수 계산 규칙과 맞춰 유지합니다.

2. When counting starts and stops2. 집계는 언제 시작하고 끝나는가

Measurement starts only after the visible countdown finishes. That avoids mixing anticipation and pre-start movement into the score. Once the countdown ends, the page counts inputs over the selected duration such as 1, 5, 10, or 60 seconds.

측정은 보이는 카운트다운이 끝난 뒤에만 시작됩니다. 그래야 시작 전에 미리 준비한 동작이나 예상 반응이 점수에 섞이지 않습니다. 카운트다운이 끝나면 1초, 5초, 10초, 60초 같은 선택 시간 동안 입력을 셉니다.

The remaining-time display uses the browser's high-resolution timing APIs, but browser timing is still affected by the task queue and device load. That is why the score is best used for repeatable self-comparison, not for universal ranking across all hardware.

남은 시간 표시는 브라우저의 고해상도 타이밍 API를 사용하지만, 브라우저 타이밍도 작업 큐와 기기 부하의 영향을 받습니다. 그래서 이 점수는 모든 하드웨어를 가르는 절대 순위보다, 같은 환경에서의 반복 비교에 더 적합합니다.

3. Why the score is useful3. 이 점수가 왜 유용한가

A CPS score is useful because it gives one consistent rule that can be repeated under matching conditions. If the device, browser, posture, and mode stay fixed, the score becomes a practical way to compare your own runs and notice rhythm or fatigue changes.

CPS 점수가 유용한 이유는 같은 조건에서 반복할 수 있는 일관된 규칙을 주기 때문입니다. 기기, 브라우저, 자세, 모드를 고정하면 자기 자신의 기록을 비교하고 리듬이나 피로 변화를 알아보는 실용적인 기준이 됩니다.

4. What the score does not claim4. 이 점수가 주장하지 않는 것

Claim주장 Why it is avoided피하는 이유
This is a universal click-speed rank.이 숫자가 절대적인 클릭 속도 순위다. Different devices, browsers, and timing conditions move the result.기기, 브라우저, 타이밍 조건이 결과를 바꿀 수 있기 때문입니다.
Mouse and touch scores are directly equal.마우스와 터치 점수는 바로 동등 비교할 수 있다. The input path and latency profile are too different.입력 경로와 지연 특성이 너무 다르기 때문입니다.
One run tells the whole story.한 번의 기록이면 충분하다. Short runs are boundary-sensitive and long runs expose fatigue.짧은 기록은 경계 시간에 민감하고, 긴 기록은 피로를 크게 드러냅니다.

5. Best practice5. 권장 사용법

  • Use the same mode when you want a fair before-versus-after comparison.
  • 전후 비교를 할 때는 같은 모드를 사용하세요.
  • Keep the device, browser, and posture fixed.
  • 기기, 브라우저, 자세를 고정하세요.
  • Prefer a small set of repeated attempts over one lucky peak score.
  • 운 좋게 나온 최고점 하나보다 반복 시도의 묶음을 보세요.
  • Read total clicks, duration, and CPS together.
  • 총 클릭 수, 시간, CPS를 함께 보세요.
